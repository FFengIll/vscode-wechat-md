## Overview

VSCode extension for WeChat Markdown formatting with local preview and one-click copy for WeChat Official Account editor. Converts Markdown to WeChat-compatible HTML with inline styles.

## Current Architecture

### Core Components

| Component | Location | Purpose |
|-----------|----------|---------|
| **Extension** | [extension.ts](src/extension.ts) | VSCode extension entry point, registers preview command |
| **PreviewPanel** | [panel/PreviewPanel.ts](src/panel/PreviewPanel.ts) | Webview panel for preview and rich copy functionality |
| **WeChatRenderer** | [renderer/index.ts](src/renderer/index.ts) | Main rendering engine with dual-mode support |
| **Rules** | [renderer/rules.ts](src/renderer/rules.ts) | Markdown-it rendering rules for WeChat compatibility |
| **Theme** | [renderer/theme.ts](src/renderer/theme.ts) | Theme system with CSS variable support |

### Dependencies

- **markdown-it**: Markdown parsing and rendering
- **@shikijs/markdown-it**: Syntax highlighting (preview mode only)
- **@shikijs/transformers**: Custom transformers for code blocks
- **VSCode Extension API**: Webview panels, file system access

### Data Flow

```
Markdown File → PreviewPanel → WeChatRenderer → markdown-it → rules → HTML Output
                                                 ↓
                                            Theme System
```

## Current Implementation Analysis

### HTML Block Handling

**Status**: ❌ NOT IMPLEMENTED

Current implementation uses `html: false` in markdown-it configuration:
```typescript
new MarkdownIt({ html: false, linkify: true, typographer: true })
```

This means:
- Raw HTML in markdown is **escaped/ignored**
- No HTML block support in preview or copy modes
- Missing feature requested in analysis report

### Code Block Handling

**Status**: ✅ PARTIALLY IMPLEMENTED

Two rendering modes:

#### Preview Mode
- Uses `@shikijs/markdown-it` plugin for syntax highlighting
- Custom transformer adds line numbers and language labels
- Output: Standard Shiki HTML with custom classes
- CSS styling via preview panel stylesheet

#### Copy Mode
- **Custom WeChat-compatible structure** (from analysis report)
- Output format:
```html
<section class="code-snippet__js">
  <pre class="code-snippet__js code-snippet code-snippet_nowrap" data-lang="lang">
    <code><span leaf="">line content</span></code>
  </pre>
</section>
```

**Issues identified**:
1. No syntax highlighting in copy mode (only plain text)
2. Missing WeChat-specific class names for coloring
3. Empty lines use `<br class="ProseMirror-trailingBreak">`

## Rendering Modes

### Preview Mode
- Goal: Local visualization with syntax highlighting
- Output: HTML with Shiki syntax highlighting
- Styles: CSS in webview panel
- Images: Local file URIs via webview API

### Copy Mode
- Goal: Direct paste into WeChat editor
- Output: WeChat-compatible HTML structure
- Styles: All inline (WeChat strips `<style>` blocks)
- Images: Base64 data URIs
- Code blocks: Custom DOM structure (no syntax highlighting)

## Theme System

**CSS Variables**: `.wechat/theme.css`
- Uses CSS custom properties (`--wechat-*`)
- Parsed on render, converted to inline styles
- Supports per-element customization (headings, colors, spacing)

**Limitations**:
- Inline styles only (no CSS selectors)
- WeChat strips external stylesheets
- Must repeat styles on each element

## Key Patterns

1. **Dual Rendering**: Separate markdown-it instances for preview/copy modes
2. **Plugin Architecture**: Shiki plugin applied only to preview mode
3. **Rule Overriding**: `applyWeChatRules()` overrides markdown-it renderers
4. **Image Resolution**: Two-phase process (webview URIs → base64 for copy)

## Integration Points

### VSCode API
- `createWebviewPanel()`: Preview panel creation
- `onDidChangeTextDocument`: Live preview updates
- `FileSystemWatcher`: Theme file hot-reload

### Webview Communication
- Messages: `copyRich`, `openTheme`, `refresh`, `doRichCopy`
- Rich copy: Two-step process (host resolves images → webview executes copy)

## Known Issues

1. **HTML blocks not supported**: Raw HTML in markdown is ignored
2. **No syntax highlighting in copy mode**: Code blocks are plain text
3. **Image loading**: Base64 conversion for copy can be slow for large files
4. **Limited HTML sanitization**: No XSS protection for user-provided HTML

## Technical Debt

- No testing infrastructure
- No error handling for malformed markdown
- Hardcoded theme defaults in TypeScript
- Complex image resolution logic (three different URI formats)
